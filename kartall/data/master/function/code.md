# 개요
- 해당 프롬포트는 master의 게임 관리 시스템 추가에 대한 가이드 프롬포트입니다.

# 설명 및 주요 기능 
- master는 기본적으로 단일 맵을 주행하는 게 아니라, 랜덤하게 뽑힌 3개의 트랙을 전부 주행하여 통과해야 하는 시스템입니다.
- 지금까지 만들어진 시스템은 단일 맵을 주행하는 시스템으로, 이제 이걸 기반으로 3개의 트랙을 주행하는 시스템으로 확장해야 합니다.
- 해당 기능은 별도의 폴더로 분리되어, function/manager 폴더에 작업물들이 들어가게 됩니다.
## 알고리즘
1. 플레이어가 마스터 시작 지점에 들어가거나 어떠한 이벤트가 발생하게 되면, 7개의 랜덤한 트랙이 풀에서 뽑히게 됩니다.
2. 뽑힌 트랙들 중 플레이어는 2개의 트랙을 밴하게 되고, 나머지 5개의 트랙 중에서 3개의 트랙이 랜덤하게 선택되어 플레이어가 주행해야 하는 트랙이 됩니다.
3. 플레이어는 선택된 3개의 트랙을 순서에 상관없이 목표 기록을 달성하여 통과해야 합니다. (시도 횟수 제한 없음)
4. 플레이어가 3개의 트랙을 전부 클리어하게 되면, 해당 3개의 트랙은 영구적으로 맵 풀에서 나오지 않게 됩니다.
5. 이후, 세 개의 맵을 모두 클리어할 시 플레이어의 masterstage가 1 상승하고, 같은 작업을 반복합니다.
## 세부 사항
### 트랙 관리 엔티티
- 트랙 관리 엔티티는 트랙의 정보를 플레이어에게 가시적으로 보여주고, 상호작용을 통해 트랙을 선택하거나 밴할 수 있도록 하는 역할을 합니다.
#### 구성 요소
- 트랙 관리 엔티티는 다음과 같은 구성 요소로 이루어집니다:
1. 트랙 이름: 트랙의 이름을 표시하는 텍스트 요소입니다.
2. 트랙 난이도: 트랙의 난이도를 표시하는 텍스트 요소입니다.
3. 트랙 목표 기록: 트랙의 목표 기록을 표시하는 텍스트 요소입니다.
4. 트랙 인터랙션: 플레이어가 트랙을 선택하거나 밴할 수 있도록 하는 인터랙션 요소입니다. (예: 버튼, 레버 등)

임시로 구현한 manager/summon-map-data/main.mcfunction 파일을 참고하여 트랙 관리 엔티티의 구현을 해야 합니다.
해당 함수는 트랙 관리 엔티티를 소환하는 함수이므로, 이 함수를 기반으로 트랙 관리 엔티티의 구성 요소들을 추가적으로 구현해야 합니다.
